# Ship Approval — claude/cool-wilson-2cb602 (local-scheduler + Crexi fix)

## Intent

Unblock Deal Hound MVP shipping. Two coupled changes that together fix the
"Crexi returns 0 results" symptom that's been blocking us for ~4 weeks and
replace the PM2-polled `worker.js` dispatch with a single-shot local
scheduler that runs all overdue buy boxes per tick, sends Resend
notification emails on completion, and is wired to a LaunchAgent for hourly
execution from Gideon's laptop.

The Crexi bug was misdiagnosed for weeks as a scraping / bot-detection
issue. Investigation confirmed Crexi is actually scraping fine (60 listings
landed today). The bug is in the skill repo's `pipeline.py` — the
`normalize_locations_to_abbrevs` function returned `set()` for compound
location names like "East Texas", causing all 60 Crexi listings to be
filtered out with `location_mismatch:TX` before scoring. That fix is on
the skills repo (`~/.claude/skills/find-deals/pipeline.py`, commit
`758cc6f`) and is not part of this PR — but is the load-bearing
dependency for this PR to deliver Crexi results to users.

The new local scheduler keeps per-user scoring (one `claude -p /find-deals
full` per overdue box) to preserve AI scoring fidelity (headed PTY mode is
required to avoid the 31% scoring divergence we hit with headless `-p`).
True group scrape optimization (one shared scrape phase + N filter+score
passes) is deferred — it requires splitting the skill into `/find-deals
scrape` and `/find-deals score` subcommands, which is a separate refactor.

## Files changed

### Worker — group scan runner (new architecture)

- `worker/group-scan-runner.js` — **new, 330+ lines.** Single-shot
  Node script (not a daemon). Reads all active `buy_boxes`, filters to
  overdue by tier interval (reusing `buy-box-scheduler.js`'s tier
  constants), creates one `deal_searches` + `scrape_jobs` row per
  overdue box, spawns one `claude -p '/find-deals full'` per box (via
  PTY runner, headed, `--dangerously-skip-permissions`), sends Resend
  notification after each, stamps `buy_boxes.last_scanned_at`. Supports
  `DEALHOUND_DRY_RUN=true` for safe testing. Includes header comment
  documenting installation, manual testing, and uninstall steps.

- `worker/union-buy-box.js` — refactored to query `buy_boxes WHERE
status = 'active'` instead of the legacy `deal_searches` table.
  Returns both the merged union AND the per-user record array so
  callers can use the union for scraping and the per-user list for
  filtering+scoring.

- `worker/email-sender.js` — added `sendScheduledScanCompleteEmail()`
  paralleling `sendFreeScanCompleteEmail()`. Same Resend client, same
  test-mode redirect, same graceful no-op when `RESEND_API_KEY` is
  missing.

- `emails/scheduled-scan-complete.js` — **new.** Resend template
  (HTML + plain text + subject) for the hourly scheduled-scan email.
  Copy: "Your Deal Hound scan is ready" with dashboard link.

- `worker/com.dealhound.group-scan.plist` — **new.** macOS LaunchAgent
  manifest. Runs `node worker/group-scan-runner.js` every 3600s with
  `RunAtLoad=false` and `KeepAlive=false`. Hardcodes
  `/opt/homebrew/bin/node` (verified present on Gideon's machine).
  stdout/stderr to `worker/logs/group-scan-runner.{out,err}.log`.

- `worker/package.json` — added `resend ^4.8.0` to dependencies (used
  by email-sender). Added `group-scan` npm script for manual triggering.

- `worker/package-lock.json` — regenerated by `npm install resend`.

### Repo hygiene

- `.gitignore` — added `worker/logs/` to exclude local log files
  written by the group-scan-runner.

## Confirmation

No files outside the intended scope were modified. The existing
`worker.js` PM2 polling daemon is left untouched and continues to
function (this PR is additive — Gideon can run the new

this dealhound-pro PR but is required for the Crexi fix to take effect.
